## Техническое задание
### Функциональные требования
- **Аутентификация**: регистрация, вход, выход, восстановление пароля
- **Профиль**: отображение, редактирование, подписки
- **Публикации**: загрузка, просмотр, удаление
- **Комментарии и лайки**: взаимодействие с постами
### Технический стек
- **Frontend**: Next.js 15, CSS, TypeScript, Zustand
- **Backend**: Next.js 15, PostgreSQL, Prisma
- **Хостинг**: Vercel
- **Хранилище**: S3
### Безопасность
- Хэширование паролей (bcrypt)
- JWT для аутентификации
### Дизайн и UI
- Минималистичный интерфейс
- Адаптивная верстка

## Примерное API
### Аутентификация
- POST /auth/register – регистрация пользователя
- POST /auth/login – вход в систему
- POST /auth/reset-password – смена пароля
- POST /auth/logout – выход из системы
- GET /auth/me – получение текущего пользователя
### Профиль
- GET /users/:id – получение данных пользователя
- PUT /users/:id – обновление профиля
- GET /users/:id/posts – получение постов пользователя
### Публикации
- POST /posts – загрузка фото
- GET /posts/:id – просмотр фото
- DELETE /posts/:id – удаление фото
### Подписки
- POST /follow/:id – подписаться на пользователя
- DELETE /follow/:id – отписаться от пользователя
- GET /users/:id/followers – получение подписчиков
- GET /users/:id/following – получение подписок
- GET /users/:id/follow-requests – получить список зпросов на подписку
- POST /follow/:id/accept – принять запрос на подписку
- DELETE /follow/:id/reject – отклонить запрос на подписку
### Лайки
- POST /posts/:id/like – лайкнуть пост
- DELETE /posts/:id/like – убрать лайк
- GET /posts/:id/likes – получить список пользователей, лайкнувших пост
### Комментарии
- POST /posts/:id/comment – оставить комментарий
- DELETE /comments/:id – удалить комментарий
- GET /posts/:id/comments – получить список комментариев под постом
### Поиск
- GET /search?query=:text – поиск по пользователям
### Лента
- GET /feed – получение списка постов от подписанных пользователей (отсортировано по дате)

## Пользовательские истории
### Регистрация и вход
Сценарий: новый пользователь хочет зарегистрироваться и войти в систему
1. Открывает страницу регистрации
2. Вводит email, имя пользователя, пароль и подтверждение пароля
3. Нажимает "Зарегистрироваться"
4. Если данные корректны, система создает аккаунт и перенаправляет на страницу входа
5. Пользователь вводит email и пароль
6. Нажимает "Войти"
7. Если данные верны, система авторизует пользователя и перенаправляет в профиль

Альтернативные сценарии:
1. Ввод некорректного email/пароля – отображение ошибки
2. Пользователь забывает пароль – использует восстановление через email

### Настройка профиля
Сценарий: пользователь хочет изменить информацию о себе
1. Переходит в свой профиль
2. Нажимает "Редактировать профиль"
3. Меняет автараку/имя/описание/email
4. Нажимает "Сохранить"
5. Данные обновляются, редирект на профиль

### Публикация фотографии
1. Нажимает кнопку "Добавить пост"
2. Выбирает изображение с устройства
3. Видит предосмотр изображения
4. Нажимает "Опубликовать"
5. Пост появляется в профиле и ленте подписчиков

### Просмотр профилей
Сценарий: пользователь хочет посмотреть профиль другого человека
1. Открывает поиск, вводит имя пользователя
2. Переходит в профиль найденного пользователя
3. Видит аватарку, описание, количество подписчиков, количество подписок, количество постов и сами посты
4. Может подписаться или отписаться
5. Листает фотографии в профиле

Альтернативный сценарий:
1. Если профиль закрытый – видит только аватарку и кнопку подписаться

### Подписки и подписчики
Сценарий: пользователь хочет подписаться на другого человека
1. Открывает профиль пользователя
2. Нажимает кнопку "Подписаться"
3. Человек появляется в списке подписок
4. Его посты появляются в ленте

### Лайк и комментарий к посту
Сценарий: пользователь хочет поставит лайк и оставить комментарий
1. Находится в профиле другого пользователя
2. Нажимает на иконку "Лайк" – количество лайков увеличивается на один
3. Открывает комментарии к посту
4. Видит другие комментарии
5. Вводит текст своего комментария
6. Нажимает "Отправить"
7. Комментарий появляется под постом

Альтернативные сценарии:
1. Нажатие на лайк повторно – количество лайков уменьшается на один

### Удаление своего поста
Сценарий: пользователь хочет удалить свою публикацю
1. Открывает свой пост
2. Нажимает на кнопку меню настроек (три вертикальные точки)
3. Выбирает "Удалить пост" 
4. Подтверждает удаление
5. Пост исчезает из профиля

### Поиск пользователей
Сценарий: пользователь хочет найти других пользователей
1.	Переходит в раздел “Поиск”
2.	Вводит имя пользователя
3.	Видит список совпадений
4.	Кликает на нужного пользователя и переходит в его профиль

### Выход из системы
Сценарий: пользователь хочет выйти из аккаунта
1.	Открывает настройки
2.	Нажимает “Выйти”
3.	Аккаунт разлогинивается, пользователя перенаправляет на страницу входа
